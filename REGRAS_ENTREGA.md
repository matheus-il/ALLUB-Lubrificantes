# ⚠️ Regra de Entrega Obrigatória - Lucy

Esta regra foi estabelecida pelo Pedrin e deve ser seguida à risca em **toda e qualquer aplicação desenvolvida ou inicializada**.

---

## 📌 A Regra de Ouro

> [!IMPORTANT]
> **Sempre que iniciarmos ou criarmos uma nova aplicação, é ESTRITAMENTE OBRIGATÓRIO desenvolver, salvar no projeto e apresentar na entrega:**
> 1. Um **Manual de Operação e Arquitetura** (`MANUAL.md`).
> 2. **Diagramas Visuais em formato Mermaid** (contendo no mínimo o fluxo de navegação do usuário e o modelo de banco de dados/entidade-relacionamento).

---

## 📋 Requisitos do MANUAL.md

O manual do projeto deve conter:
- **Guia de Instalação e Execução**: Instruções claras de comandos (`npm install`, `npm run dev`, migrations, etc.) para rodar o projeto localmente.
- **Configurações Necessárias**: Detalhamento de variáveis de ambiente (`.env.example`) e chaves secretas necessárias.
- **Diagrama de Fluxo (Mermaid)**: Mapeamento visual das telas, modais e transições que o usuário realiza.
- **Diagrama de Banco de Dados (Mermaid)**: Modelo Entidade-Relacionamento (ERD) representando as tabelas, campos e chaves estrangeiras.
- **Guia de Validação/Build**: Comandos necessários para rodar testes unitários, formatações (lint) e empacotamento de produção.
